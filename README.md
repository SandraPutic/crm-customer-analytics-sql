# CRM Customer Analytics (SQL Project)

##  Описание проекта
В данном проекте проведён анализ клиентской базы CRM с использованием SQL с целью изучения поведения пользователей, их удержания и сегментации.

## Цели анализа
- Определить активных и оттокших клиентов  
- Рассчитать уровень удержания (retention)  
- Сегментировать клиентов по ценности  
- Проанализировать покупательское поведение  

## Описание данных
Использованы две таблицы:
- customers (customer_id, signup_date) — информация о клиентах  
- orders (order_id, customer_id, order_date, amount) — информация о заказах  

## Используемые методы и навыки
- JOIN-запросы  
- Агрегации (SUM, COUNT, AVG)  
- GROUP BY  
- Оконные функции (LAG, RANK)  
- CTE (WITH)  

## Ключевые выводы
- Значительная часть пользователей совершает только одну покупку, что указывает на риск оттока  
- Небольшая доля клиентов (VIP) генерирует основную часть выручки  
- У некоторых клиентов наблюдаются длительные интервалы между покупками  

## Бизнес-рекомендации
- Запустить кампании по удержанию клиентов после первой покупки  
- Сфокусироваться на работе с VIP-сегментом  
- Использовать персонализированные предложения для “спящих” клиентов
